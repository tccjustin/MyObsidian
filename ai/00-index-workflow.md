# 00-index 노트 정리 워크플로우

## 기본 워크플로우
1. 사용자가 `00-index` 폴더에 노트를 작성
2. 사용자가 "이 내용을 따로 정리해줘" 또는 "이 내용을 [폴더명]에 정리해줘"라고 지시
3. AI가 내용을 분석하여 적절한 폴더를 결정하고 파일을 정리

## 자동 분류 규칙

### 키워드 기반 폴더 매핑

#### 회의록/미팅 관련: `01-회의록`
- 키워드: 회의, 미팅, meeting, 회의록, 워크숍, 세미나, 리뷰, 협의
- 파일명 예시: `YYYY-MM-DD-[주제].md` 형식 권장

#### 개발팁/기술문서: `03-개발팁`
- 키워드: 개발, 팁, tip, 방법, 해결, 설정, 구성, 사용법, 가이드
- 기술 관련 내용 (Yocto, Linux, Git, VSCode 등)

#### 프로젝트별 폴더 (주제가 명확한 경우)
- `autolinux`: autolinux, build-autolinux 관련
- `tcs`: TCS, Jira, 워크로그, OKR 관련
- `wiki`: 위키, 문서화 관련
- `chatgpt`: ChatGPT 대화 기록 (날짜 형식: `YYYY-MM-DD-[주제].md`)

#### 새 폴더 생성 (기존 폴더에 맞지 않는 경우)
- 주제가 명확하면 새 폴더 생성 (예: `02-[주제명]`)
- 폴더명은 소문자, 하이픈 사용 (예: `02-test-system`)

## 작업 절차

1. **파일 읽기**: `00-index` 폴더의 해당 파일 읽기
2. **내용 분석**: 파일 내용과 파일명을 분석하여 키워드 추출
3. **폴더 결정**:
   - 사용자가 명시적으로 폴더를 지정한 경우: 해당 폴더 사용
   - 자동 분류: 키워드 매핑 테이블 기반으로 결정
   - 불명확한 경우: 사용자에게 확인 요청
4. **폴더 생성**: 폴더가 없으면 생성
5. **파일 이동/복사**:
   - 기본적으로 파일을 대상 폴더로 **이동** (원본 삭제)
   - 사용자가 명시적으로 요청하면 복사 가능
6. **파일명 정리**:
   - 필요시 파일명을 더 명확하게 변경 (예: 날짜 추가, 주제 명확화)
   - 기존 파일명이 적절하면 유지

## 사용자 지시 예시
- "이 내용을 따로 정리해줘" → 자동 분류
- "이 내용을 회의록 폴더에 정리해줘" → `01-회의록` 폴더로 이동
- "이 내용을 tcs 폴더에 복사해줘" → `tcs` 폴더로 복사 (원본 유지)
- "이 내용을 새 폴더 만들어서 정리해줘" → 새 폴더 생성 후 이동

## 주의사항
- `00-index` 폴더는 임시 노트 작성용이므로, 정리 후에는 원본 파일을 삭제하는 것이 기본
- 파일을 이동하기 전에 사용자에게 목적지 폴더를 알려주고 확인받는 것이 좋음
- 날짜가 포함된 내용은 파일명에 날짜를 포함시키는 것을 권장
