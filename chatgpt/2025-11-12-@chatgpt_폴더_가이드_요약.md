### 안내
`chatgpt` 폴더의 각 문서를 빠르게 활용할 수 있도록 핵심만 요약했습니다. 자세한 스니펫·배경 설명은 각 원문을 참고하세요.

---

### 2025-11-05-빠른_다운로드_방법.md
- **핵심**: FTP 대량 전송은 병렬·분할·이어받기 지원 도구가 빠름.
- **추천**: `lftp mirror --parallel --use-pget-n`, 또는 큰 파일은 `aria2c`, 유지 시 `ncftpget -z`.
- **명령 예시**
  - lftp 디렉터리 동기화: `mirror --parallel=8 --use-pget-n=8 /share/kirkstone ./source-mirror`
  - aria2c 분할: `aria2c -x16 -s16 -k1M ftp://.../tools-kirkstone.tar.gz`
- **팁**: 서버 동시 접속 제한 시 8→6→4로 조절, 중단 시 자동 이어받기 활용.

### 2025-11-12-Gemini_Code_Assist_오류_해결.md
- **증상**: `GLIBC_ABI_DT_RELR` 미존재로 GCA 서버 크래시 → glibc 2.36+ 필요.
- **해결**: (A) OS/WSL을 Ubuntu 24.04 등으로 업그레이드, (B) GCA 다운그레이드, (C) Dev Container 사용.
- **확인**: `ldd --version` 결과 2.36+인지 확인.
- **캐시 초기화**: `rm -rf ~/.cache/cloud-code` 후 재시작.

### 2025-11-12-레시피_찾기_방법.md
- **devtool 검색 순서**: devtool workspace → `bblayers.conf`의 레이어들 → bitbake 캐시.
- **없을 때 확인**
  - `devtool status`
  - `bitbake-layers show-recipes | grep <name>`
  - `find ../ -name "<name>.bb"`
- **workspace_path**: 기본 1개만 유효. 멀티 필요 시 별도 build-dir or 심볼릭 링크로 우회.
- **sstate 로그 해석**: setscene 캐시 매칭률, stale 제거, 이후 task 실행 흐름 확인.

### 2025-11-12-VSCode_폴더_숨기기.md / (1).md
- **Repositories 숨기기 방법**
  - 가장 확실: 대상 폴더의 `.git` 이름 변경(복원 시 되돌림).
  - 설정식: `"git.ignoredRepositories"`(최상위 repo만), 멀티루트에선 폴더 별 `"git.enabled": false`.
  - 자동 스캔 끄기: `"git.scanRepositories": false` → 하위 repo 자동감지 차단.
  - UI만 숨김: `"files.exclude"`, `"search.exclude"`.
- **주의**: `git.repositoryScanIgnoredFolders`는 절대경로 아님(이름/글롭 사용: `"**/build-axon"`).

### 2025-10-29-Conditional_npm_install.md / (1).md
- **목표**: `precompile`에서 불필요한 `npm install` 방지.
- **옵션 A(원라이너)**: `package-lock.json` 해시를 `node_modules/.depshash`와 비교해 달라졌을 때만 설치.
- **옵션 B(스크립트 파일)**: `scripts/ensure-deps.js`로 해시 비교 후 `npm ci` 실행.
- **권장**: CI/빌드는 `npm ci --prefer-offline --no-audit --progress=false`.

### 2025-11-12-IPSec_테스트_설정.md
- **구성**: 192.168.0.10 ↔ 192.168.0.1, ESP/transport, `auth hmac(md5)`, `enc cbc(aes)`, 키 128bit(0x22…22).
- **스크립트**: `setup_a.sh / setup_b.sh`, 정리용 `cleanup_*.sh`.
- **검증**: `ip xfrm state/policy`, `tcpdump -ni <if> proto 50`, `ping` 왕복 확인.
- **주의**: FIPS 모드에서 MD5 비활성화 가능.

### 2025-11-05-Bosch_ROM_및_RS-CAN_지원.md
- **ROM(Restricted Operation Mode)**: 수신/ACK만, 송신·에러/오버로드 프레임 미전송, 오류 카운터 제한.
- **Renesas**: RS-CANFD에서 CAN FD 모드에 한해 유사 기능 문서화. 클래식 CAN은 문서상 불명확.
- **RAM Watchdog**: Bosch IP는 존재(타임아웃 인터럽트). Renesas 문서에서 동등 기능은 명확치 않음(ECC·초기화 절차는 별도 존재).

---

### 2025-11-05-원격_브랜치_push_문제.md
- **non-fast-forward**: 원격이 앞서면 push 거부 → `git fetch` 후 `rebase` 또는 `merge`.
- **안전 강제푸시**: 개인 브랜치 한정 `--force-with-lease`.
- **Jira 훅 거부**: 커밋 메시지에 이슈키 포함(예: `SD710XL-1670:`). 일괄 수정은 `git rebase -i`.
- **안전 흐름**
  - `git fetch --all`
  - `git rebase origin/main` 또는 `git merge origin/main`
  - 최초 업스트림: `git push -u origin <branch>`
  - 재작성 후: `git push --force-with-lease`(필요 시)
