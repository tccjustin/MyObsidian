## You

vscode에서 특정 폴더가 source control에서 안 보이게 할 수 있나? 구체적으로는 repositories 목록에서 안 나오게 하고 싶어..

---

## ChatGPT

좋은 질문이에요 👍  
Visual Studio Code(특히 WSL 환경 포함)에서 **Source Control 뷰의 “Repositories” 목록에서 특정 폴더(리포지토리)를 숨기는 방법**은 기본적으로 “해당 폴더가 Git 리포지토리로 감지되지 않도록” 처리하는 방식으로 해결해야 합니다.  

---

### ✅ 방법 1: `.git` 폴더를 숨기기 (가장 확실)
VSCode는 폴더 내 `.git` 디렉토리가 있으면 자동으로 “repository”로 인식합니다.  
→ 따라서 이 폴더가 감지되지 않게 하면 됩니다.

#### 방법:
1. 숨기고 싶은 폴더의 `.git` 폴더명을 변경  
   ```bash
   mv path/to/target/.git path/to/target/.git_backup
   ```
2. 이렇게 하면 VSCode는 해당 폴더를 Git 저장소로 인식하지 않으므로 “Source Control → Repositories” 목록에서 사라집니다.
3. 다시 활성화하려면:
   ```bash
   mv path/to/target/.git_backup path/to/target/.git
   ```

> 💡 단점: Git 명령어도 그 폴더 내에서는 동작하지 않습니다. (즉, 완전히 Git 비활성화)

---

### ✅ 방법 2: `settings.json`에서 Git 감지 제외
VSCode의 설정에서 **특정 폴더를 Git 자동 감지에서 제외**할 수 있습니다.

#### 설정 파일:
`settings.json` (User 또는 Workspace 수준)

#### 추가 내용:
```json
"git.ignoredRepositories": [
    "/home/username/work/projectA",
    "/mnt/c/Users/.../work/projectB"
]
```

> ⚠️ 단, `git.ignoredRepositories` 설정은 **VSCode Insiders** 또는 **Git extension 버전 1.95 이상**에서만 동작하는 경우가 있습니다. (안 되면 아래 방법 3으로)

---

### ✅ 방법 3: `files.exclude`로 UI에서 숨기기
단순히 **탐색기와 Source Control에서 시각적으로 안 보이게** 하고 싶을 때:

```json
"files.exclude": {
    "**/target-folder-name": true
},
"search.exclude": {
    "**/target-folder-name": true
}
```

> 이렇게 하면 폴더가 탐색기나 검색 결과에서도 안 보이지만, Git 자체는 여전히 감지합니다.  
> 즉, Source Control 뷰에는 여전히 repository로 나타날 수 있습니다.

---

### ✅ 방법 4: Workspace에서 제외
만약 **멀티-루트 Workspace**(`.code-workspace` 파일을 사용 중이라면):
```json
"folders": [
  {
    "path": "repo-main"
  },
  {
    "path": "repo-dev",
    "name": "repo-dev (hidden)",
    "settings": {
      "git.enabled": false
    }
  }
]
```

이렇게 `git.enabled: false`를 지정하면 해당 폴더만 Git 관리에서 제외됩니다.

---

### ✅ 추천 정리

| 목적 | 추천 방법 | 비고 |
|------|-------------|------|
| 완전히 repository 목록에서 제거 | `.git` 폴더 이름 변경 | Git 명령도 비활성화됨 |
| 일부 폴더만 VSCode에서 Git 감지 제외 | `git.ignoredRepositories` 또는 workspace 설정에서 `"git.enabled": false` | 최신 VSCode 필요 |
| 단순히 UI에서 안 보이게 | `files.exclude` | Git 관리에는 여전히 포함됨 |

---

아주 좋은 포인트 짚으셨어요 👏  
`"git.ignoredRepositories"` 옵션은 직관적으로 보이지만, 실제로는 **“최상위 리포지토리 경로만”** 무시합니다.  
즉, `build-axon` **하위에 있는 여러 개의 Git repo들**(`build-axon/foo/.git`, `build-axon/bar/.git` 등)은 여전히 각각 별도 리포지토리로 감지됩니다.

---

### 🔍 원인 정리
VSCode의 Git 확장은 다음과 같이 작동합니다:
- 폴더 트리를 훑으면서 `.git` 디렉토리가 있으면 **각각 별도의 repository**로 등록합니다.
- `"git.ignoredRepositories"`는 **그 경로에 직접 `.git`이 존재하는 폴더**만 무시합니다.
  - 즉, `/build-axon` 자체가 Git repo일 경우에만 제외됨.
  - 하지만 `/build-axon/sub1/.git`, `/build-axon/sub2/.git`은 따로 감지됨.

---

### ✅ 해결 방법 1 — 하위 경로를 전부 명시 (간단하지만 명시적)
가장 직접적인 방법은 **하위 리포지토리들을 모두 지정**하는 것입니다.

예를 들어 아래 구조라면:
```
build-axon/
 ├── repo1/.git
 ├── repo2/.git
 └── repo3/.git
```

이렇게 설정해야 합니다:
```json
"git.ignoredRepositories": [
    "${workspaceFolder}/build-axon/repo1",
    "${workspaceFolder}/build-axon/repo2",
    "${workspaceFolder}/build-axon/repo3"
]
```

단점: 하위 폴더를 추가할 때마다 직접 추가해야 함.

---

### ✅ 해결 방법 2 — `git.scanRepositories` 설정 끄기 + 상위 폴더만 감시
VSCode는 기본적으로 **하위 폴더까지 자동으로 Git repo를 스캔**합니다.  
이를 끄면, 의도한 상위 폴더만 Repositories에 표시되게 할 수 있습니다.

```json
"git.scanRepositories": false,
```

이렇게 하면:
- VSCode는 **현재 열려 있는 workspace 폴더만 repo로 감지**
- 하위 폴더(예: build-axon 내부의 git들)는 자동으로 감지하지 않음

➡️ 이게 **“build-axon 안의 repo들만 숨기고 싶다”**일 때 제일 깔끔한 방법입니다.

---

### ✅ 해결 방법 3 — Workspace 설정에서 해당 폴더만 Git 끄기
만약 `build-axon` 폴더가 **workspace에 포함된 루트**라면,  
그 폴더에만 다음 설정을 줄 수 있습니다:

`.vscode/settings.json` (혹은 `.code-workspace` 파일 내):
```json
{
  "folders": [
    { "path": "src" },
    { "path": "build-axon", "settings": { "git.enabled": false } }
  ]
}
```

이러면 `build-axon` 아래 리포지토리는 전부 VSCode Git에서 비활성화됩니다.

---

### ✅ 해결 방법 4 — 강제적으로 Git 폴더 숨기기 (마지막 수단)
아예 감지가 안 되게 `.git` 폴더를 숨기는 방법입니다:

```bash
find build-axon -name ".git" -type d -exec mv {} {}_hidden \;
```

이러면 Source Control에서 싹 사라집니다.  
필요할 때 다시 복원:

```bash
find build-axon -name ".git_hidden" -type d -exec bash -c 'mv "$0" "${0%_hidden}"' {} \;
```

| 목표 | 추천 방법 | 설명 |
|------|-------------|------|
| 하위 repo 전부 숨기기 | `"git.scanRepositories": false` | 자동 스캔 비활성화 |
| 특정 repo만 숨기기 | `"git.ignoredRepositories": [...]` | 경로 하나씩 지정 |

VS Code의  
```json
"git.ignoredRepositories": [ ... ]
```
설정은 현재까지 동작 방식상 **상대 경로(`"${workspaceFolder}/..."`나 `./subdir`)를 신뢰성 있게 처리하지 못하고**,  
**절대 경로만 확실하게 인식**하는 게 맞습니다.

---

### 🔍 왜 그런가?
VS Code의 Git 확장은 내부적으로 이렇게 동작합니다:

1. Git 저장소를 스캔할 때,  
   각 repository의 **절대 경로(absolute path)** 를 저장합니다.  
   (예: `/home/B030240/work/.../repo1`)
2. `git.ignoredRepositories` 목록에 포함된 경로와 **문자열 비교**를 합니다.
   → 상대 경로나 `${workspaceFolder}` 변수는 단순 문자열로 처리되어 **확장되지 않거나 매칭 실패**합니다.

그래서  
```json
"git.ignoredRepositories": [
    "${workspaceFolder}/build-axon"
]
```
은 실제로는  
`/home/B030240/work/.../build-axon` 이라는 경로와 **매칭되지 않습니다.**

---

### ✅ 올바르게 동작시키려면
절대 경로를 직접 지정해야 합니다:

```json
"git.ignoredRepositories": [
    "/home/B030240/work1/ext-test/test-devtool2/build-axon"
]
```

> 이러면 확실하게 repository 목록에서 사라집니다.

---

경로가 자주 바뀌는 환경이라면, **workspace 설정 대신 User-level 설정**에 등록하는 게 낫습니다.  
또는 `.code-workspace` 파일을 쓰는 경우 `"path"`가 절대경로로 지정되므로  
`${workspaceFolder}` 대신 `"git.enabled": false` 조합을 쓰는 편이 더 안정적입니다.

예:
```json
{
  "folders": [
    { "path": "." },

```

| `${workspaceFolder}` | ⚠️ 인식 안 됨 | 문자열 그대로 처리됨 |

## ✅ 핵심 개념
VS Code는 기본적으로 **모든 하위 폴더를 재귀적으로 스캔해서 `.git` 폴더를 찾습니다.**  
즉, 여러 Git repo가 있으면 전부 *Repositories* 목록에 뜨죠.

이걸 제어하는 방법은 아래 두 가지 패턴 중 하나입니다:

---

## 🧩 방법 1 — “자동 스캔” 꺼두고 수동으로 지정하기
### 👉 `"git.scanRepositories": false`
이 옵션을 끄면,  
VS Code는 더 이상 하위 폴더를 자동으로 Git repository로 인식하지 않습니다.

즉, **현재 열려 있는 루트 폴더**만 Source Control에 표시됩니다.

```json
"git.scanRepositories": false
```

➡️ 그 상태에서 **보고 싶은 repo만 Workspace에 직접 포함**시키면 됩니다.

### 예시: `.code-workspace` 구성
```json
{
  "folders": [
    { "path": "repo-main" },
    { "path": "tools/common-lib" }
  ],
  "settings": {
    "git.scanRepositories": false
  }
}
```

이러면 VSCode는:
- `repo-main` 과 `tools/common-lib` 만 “Repositories”에 표시됨  
- 그 외의 Git repo들은 전혀 안 보임  

💡 *가장 깔끔하고 확실한 방식이에요.*

---

## 🧩 방법 2 — `"git.repositoryScanDepth"` 제한하기
자동 스캔을 완전히 끄기 부담스럽다면,  
깊이 제한을 줄 수 있습니다.

```json
"git.repositoryScanDepth": 1
```

이러면 현재 workspace 폴더 바로 아래까지만 Git repo를 감지합니다.  
즉, 2단계 이상 하위 폴더의 repo들은 무시됩니다.

---

## 🧩 방법 3 — `"git.enabled": false` 로 개별 repo 끄기
멀티-root workspace를 쓰고 있다면  
특정 폴더만 Git 기능을 꺼버릴 수 있습니다.

예시:
```json
{
  "folders": [
    { "path": "repo-core" },
    { "path": "repo-experimental", "settings": { "git.enabled": false } },
    { "path": "repo-driver" }
  ]
}
```

이러면:
- `repo-core` ✅ 표시됨  
- `repo-driver` ✅ 표시됨  
- `repo-experimental` ❌ 감지 안 됨  

---

## 🧩 방법 4 — 완전 수동 제어 (고급)
VSCode는 내부적으로 감지된 모든 repo 경로를 저장합니다.  
`"git.ignoredRepositories"`를 절대경로 기반으로 쓰면  
화이트리스트처럼 작동시킬 수도 있습니다.

예시:
```json
"git.scanRepositories": true,
"git.ignoredRepositories": [
    "/home/B030240/work1/ext-test/build-axon/repo1",
    "/home/B030240/work1/ext-test/build-axon/repo2",
    "/home/B030240/work1/ext-test/build-axon/repo3"
]
```
👉 **표시하고 싶은 repo만 제외한 나머지를 전부 ignoredRepositories에 추가**  
(관리하기는 조금 번거로움)

## ✅ 핵심 설정: `git.repositoryScanIgnoredFolders`

이 설정이 바로 **VSCode가 Git 저장소를 자동으로 스캔할 때 제외할 경로 목록**을 지정하는 기능입니다.

---

### 📘 사용 예시

`settings.json` (User 또는 Workspace 설정)에 아래처럼 추가합니다 👇

```json
"git.repositoryScanIgnoredFolders": [
    "/home/B030240/work1/ext-test/test-devtool2/build-axon"
]
```

---

### 💡 동작 방식
- VSCode는 기본적으로 **모든 하위 폴더를 재귀적으로 스캔**하여 `.git` 디렉터리가 있으면 Source Control에 표시합니다.
- 위 설정을 추가하면, 지정한 폴더(및 그 하위 폴더)는 **Git 스캔 자체에서 제외**됩니다.
- 따라서 `Repositories` 목록에 나타나지 않습니다.
- 다른 Git 리포지토리는 정상적으로 유지됩니다.

## ✳️ 주요 패턴 기호 정리

| 패턴 | 의미 | 예시 | 매칭 예시 |
|------|------|------|-----------|
| `*` | 슬래시(`/`)를 제외한 어떤 문자열이든 0개 이상 | `build*` | `build`, `build-axon`, `builder` |
| `?` | 임의의 한 문자 | `file?.txt` | `file1.txt`, `fileA.txt` |
| `**` | **디렉터리 포함한 모든 하위 경로** 재귀 매칭 | `**/build` | `build`, `a/build`, `a/b/build` |
| `{a,b}` | 여러 패턴 중 하나 선택 | `{build,output}` | `build`, `output` |
| `!` | 부정 (해당 패턴 제외) | `!**/keep` | `keep` 폴더만 제외하고 나머지 숨김 |

## 🧩 1️⃣ 어디에 저장되나
VS Code는 **사용자 데이터 디렉터리**(`User Data Dir`) 안에 이 정보를 저장합니다.

| OS | 저장 경로 예시 |
|----|----------------|
| **Windows** | `%APPDATA%\Code\User\workspaceStorage\` |
| **WSL / Linux** | `~/.config/Code/User/workspaceStorage/` |
| **macOS** | `~/Library/Application Support/Code/User/workspaceStorage/` |

- 각 하위 폴더(`workspaceStorage/<랜덤 UUID>/`)가 **열었던 워크스페이스 하나당** 저장소입니다.  
- 그 안에 있는 `state.vscdb` 파일(작은 SQLite DB)에  
  - 마지막으로 열린 리포지토리 목록  
  - Git 확장 관련 상태(HEAD, remotes 등)  
  - UI 상태(탭, folding, etc.)  
  등이 들어 있습니다.  

---

## 🧩 2️⃣ 무슨 정보가 저장되나
- `repositoryRoot` (리포지토리 루트 경로)  
- `repositoryUIState` (열림 / 닫힘, selected repo 등)  
- `HEAD`, `remote`, `branch` 캐시  
- 최근 커밋 SHA 캐시  
이런 값들이 JSON / SQLite 형태로 남아요.

---

## 🧩 3️⃣ 캐시 초기화 / 리셋 방법
리포지토리 목록이 꼬였거나 안 사라질 때는 다음 순서로 리셋하면 됩니다.

1. **VS Code 완전히 종료**
2. 아래 경로로 가서 `workspaceStorage` 폴더 통째로 삭제  
   ```bash
   rm -rf ~/.config/Code/User/workspaceStorage/*
   ```
3. VS Code 재시작 → 다시 스캔 시 목록 재구성

> 📌 이렇게 지워도 확장 설정 / 키바인딩 같은 건 안 지워집니다.  
> 오직 “워크스페이스 상태”만 초기화돼요.
